# ✍️ 内容创作矩阵 (Content Creation Matrix)

这是一个**多智能体（Multi-Agent）协同架构**的企业级内容创作与诊断生态系统。

## 🌟 架构说明
本项目由 **1个主调度节点 (Router)** 和 **4个垂直领域专家节点 (Sub-Agents)** 组成。它们各司其职，遵循“单一职责原则”，完美解决了大模型在复杂任务中的指令超载与幻觉问题。

### 📁 矩阵组件说明

1. **`content-matrix-router`** (总调度官)
   - 核心职责：作为唯一对外接口，识别用户的复合意图，并在后台动态调度不同的专家。
   - 工作流：负责任务分发、条件判断（如：打分不及格则拦截排版）、结果合并。

2. **`content-crafting`** (内容创作大师)
   - 核心职责：提供从0到1的构思指导。
   - 知识库：5步创作引导法（选题、标题、开头、正文、结尾）、降AI味润色法则。

3. **`content-evaluator`** (内容质检官)
   - 核心职责：对写好的草稿进行冷酷无情的体检。
   - 知识库：110分双轨制评估体系（100分基础质量 + 10分降AI味）。

4. **`content-channel-adapter`** (渠道分发专家)
   - 核心职责：根据目标平台进行格式重构与排版。
   - 知识库：公众号（深度留白）、小红书（高频Emoji+列点）、今日头条（小标题+SEO）的排版规范。

5. **`content-data-diagnoser`** (数据诊断师)
   - 核心职责：通过发布后的核心指标反向推导内容病因。
   - 知识库：打开率/完播率/分享率的归因模型与迭代药方。

## 🚀 部署与使用指南

### 在 Coze / Dify / OpenClaw 等工作流平台部署
1. 将 4 个 Sub-Agents 的 `SKILL.md` 分别创建为独立的 Agent 节点。
2. 创建一个主 Agent 节点，填入 `content-matrix-router/SKILL.md` 的提示词。
3. 通过意图识别/分类器节点，将主 Agent 与 4 个子 Agent 进行连线。

### 在 Trae IDE / Trae Solo 客户端本地使用
1. 将这 5 个文件夹完整复制到你的 `.trae/skills/`（IDE版）或 `~/.trae-cn/skills/`（Solo客户端版）目录下。
2. 在对话框中输入 `/` 唤起技能列表，选择 `content-matrix-router`。
3. 直接输入自然语言指令（例如：“帮我看看这篇草稿，及格了就排版成小红书风格”），系统将自动实现本地多 Agent 静默协同。
